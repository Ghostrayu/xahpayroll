## DEVELOPMENT SETUP

### PREREQUISITES

BEFORE YOU BEGIN, ENSURE YOU HAVE:

- **NODE.JS**: VERSION 18.0.0 OR HIGHER
- **NPM**: VERSION 9.0.0 OR HIGHER
- **POSTGRESQL**: VERSION 14 OR HIGHER
- **GIT**: FOR VERSION CONTROL

### INSTALLATION STEPS

1. **CLONE THE REPOSITORY**

   ```bash
   git clone https://github.com/YOUR-ORG/xah-payroll.git
   cd xah-payroll
   ```

2. **INSTALL DEPENDENCIES**

   ```bash
   npm run install:all
   ```

3. **SET UP ENVIRONMENT VARIABLES**

   ```bash
   # COPY ENVIRONMENT TEMPLATES
   cp frontend/.env.example frontend/.env
   cp backend/.env.example backend/.env

   # EDIT .ENV FILES WITH YOUR CONFIGURATION
   # SEE DOCUMENTS/DATABASE_SETUP.MD FOR DATABASE CONFIGURATION
   ```

4. **INITIALIZE DATABASE**

   ```bash
   cd backend
   npm run init-db
   ```

5. **START DEVELOPMENT SERVERS**

   ```bash
   # FROM PROJECT ROOT
   npm run dev
   ```

6. **VERIFY INSTALLATION**

   - FRONTEND: HTTP://LOCALHOST:3000
   - BACKEND: HTTP://LOCALHOST:3001
   - HEALTH CHECK: HTTP://LOCALHOST:3001/HEALTH

FOR DETAILED SETUP INSTRUCTIONS, SEE [DOCUMENTS/QUICKSTART.MD](DOCUMENTS/QUICKSTART.MD).

## CODE STYLE GUIDELINES

### GENERAL PRINCIPLES

- **READABILITY**: CODE SHOULD BE SELF-DOCUMENTING
- **SIMPLICITY**: PREFER SIMPLE SOLUTIONS OVER COMPLEX ONES
- **CONSISTENCY**: MATCH EXISTING CODE STYLE IN THE PROJECT
- **DRY**: DON'T REPEAT YOURSELF

### JAVASCRIPT/TYPESCRIPT

```javascript
// ‚úÖ GOOD: DESCRIPTIVE NAMES, CLEAR LOGIC
const calculateHourlyPayment = (hoursWorked, hourlyRate) => {
  return hoursWorked * hourlyRate;
};

// ‚ùå BAD: UNCLEAR NAMES, COMPLEX NESTING
const calc = (h, r) => { return h * r; };
```

**CONVENTIONS**:
- USE `CAMELCASE` FOR VARIABLES AND FUNCTIONS
- USE `PASCALCASE` FOR REACT COMPONENTS AND CLASSES
- USE `UPPERCASE` FOR CONSTANTS
- PREFER `CONST` OVER `LET`, AVOID `VAR`
- USE ARROW FUNCTIONS FOR CALLBACKS

### USER-FACING TEXT

**CRITICAL**: ALL USER-FACING TEXT MUST USE FULL CAPITALIZATION (ALL CAPS)

```javascript
// ‚úÖ CORRECT
<button>CANCEL PAYMENT CHANNEL</button>
alert('INVALID WALLET ADDRESS FORMAT');

// ‚ùå WRONG
<button>Cancel Payment Channel</button>
alert('Invalid wallet address format');
```

SEE [CLAUDE.MD](CLAUDE.MD) FOR COMPLETE TEXT CAPITALIZATION GUIDELINES.

### FILE ORGANIZATION

- **COMPONENTS**: `FRONTEND/SRC/COMPONENTS/`
- **CONTEXTS**: `FRONTEND/SRC/CONTEXTS/`
- **UTILITIES**: `FRONTEND/SRC/UTILS/`
- **API ROUTES**: `BACKEND/ROUTES/`
- **DATABASE**: `BACKEND/DATABASE/`

### IMPORTS

ORGANIZE IMPORTS IN THIS ORDER:

```javascript
// 1. EXTERNAL LIBRARIES
import React, { useState, useEffect } from 'react';
import { useNavigate } from 'react-router-dom';

// 2. INTERNAL MODULES
import { useAuth } from '@/contexts/AuthContext';
import { api } from '@/services/api';

// 3. COMPONENTS
import PaymentChannelModal from '@/components/PaymentChannelModal';

// 4. STYLES
import './styles.css';
```

## TESTING REQUIREMENTS

### BEFORE MAKING CHANGES

1. **LINT YOUR CODE**
   ```bash
   npm run lint
   ```

2. **BUILD SUCCESSFULLY**
   ```bash
   npm run build
   ```

3. **TEST DATABASE CONNECTION** (IF BACKEND CHANGES)
   ```bash
   cd backend
   npm run test:db
   ```

4. **MANUAL TESTING**
   - TEST YOUR CHANGES IN BOTH TESTNET AND MAINNET MODES
   - VERIFY WALLET INTEGRATION (XAMAN)
   - CHECK PAYMENT CHANNEL FUNCTIONALITY
   - TEST EDGE CASES AND ERROR HANDLING
